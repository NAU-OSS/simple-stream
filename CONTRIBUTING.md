# Contributing to simple-stream

Thank you for your interest in contributing to simple-stream! Contributions of all kinds are welcome, including bug reports, feature requests, documentation improvements, and code contributions.

## How to Contribute
### Issues

- Search existing issues before opening a new one.

- Use clear, descriptive titles.

- Provide enough detail to reproduce the problem or understand the request.

### Pull Requests

- Fork the repository.

- Create a new branch for your change:
```
git checkout -b feature-or-fix-name
```
- Make your changes and commit them with clear messages.

- Push your branch to your fork.

- Open a pull request against the main branch.

### Code Style and Formatting Guidelines

- Follow standard C conventions.

- Use consistent indentation (4 spaces, no tabs).

- Keep functions small and focused.

- Use descriptive variable and function names.

- Include comments where logic is non-obvious.

### Testing Requirements

- All code must compile without warnings:
```
gcc -Wall -pedantic
```
- New features should include basic test instructions or example usage.

### Documentation Standards

- Functions should be documented with helpful comments.

### How to Report Bugs

When reporting a bug, please include:

- A clear description of the issue.

- Steps to reproduce the problem.

- Bug reports should be submitted through GitHub Issues.

### How to Propose New Features

- Open a GitHub Issue labeled as a feature request.

- Describe the problem the feature solves.

### Community Guidelines and Expectations

- Be respectful and constructive.

- Welcome contributors of all experience levels.

- Provide helpful feedback during reviews.

- Harassment, discrimination, or abusive behavior will not be tolerated.

**By participating in this project, you agree to uphold these standards.**
